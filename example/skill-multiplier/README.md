# Skill Multiplier

> **Why.** The hub holds ~430 entries today, but it has always grown by human authorship. What if every skill + knowledge pair is already a latent *third* skill waiting to be synthesized? This tool runs a generational loop: Gen 0 = the live hub, every tick picks two parents (weighted by tag/category overlap, with a mutation chance), deterministically synthesizes a child `SKILL.md` draft, and drops it back into the pool as a Gen-1 entry. The next tick can pair a Gen-1 child with an original — so children beget children. Watch the population double, triple, keep going.

## Features

- Self-running loop with **Start / Pause / Reset** and a 1–20 gen/sec speed slider.
- **Mutation rate** slider (0–100%) injects a random global-pool tag into the child to prevent monocultures.
- **Lineage tree** on the middle panel — roots (Gen 0 parents) → descendants — click any node to preview its synthesized `SKILL.md` body.
- **Live event log** (left) shows every cross, color-coded by generation (Gen 0–5+).
- **Export** dumps the full lineage + bodies as JSON.
- Deterministic combiner (mulberry32 PRNG over parent name words, tag unions, category preference) — **no LLM calls**, fully offline.
- Zero-dep: pure HTML + single tiny Node static server.

## Usage

```bash
# 1. Build the corpus (from a local kjuhwa/skills-hub working copy, OR any
#    repo with .claude/skills + .claude/knowledge subfolders):
cd /path/to/skills-hub
node /path/to/skill-multiplier/browser/build-index.mjs

# 2. Point the server at it (or drop skills.json next to serve.mjs):
cd skill-multiplier/browser
CORPUS=/path/to/skills.json node serve.mjs
# \u25b6 skill multiplier \u2192 http://localhost:4177/
```

The corpus path resolution order is: `$CORPUS` env var \u2192 `./skills.json` next to `serve.mjs` \u2192 `../../browser/skills.json` (dev workspace fallback).
