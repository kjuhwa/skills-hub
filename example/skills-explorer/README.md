# Skills Explorer

> **Why.** A single working copy of `kjuhwa/skills-hub` can accumulate hundreds of skills and knowledge entries (this one has 217 + 213 = 430). Reading them one `SKILL.md` at a time doesn't scale — you need a browseable catalog with fuzzy search. This tool turns any local checkout of the hub into an instantly-searchable dashboard, zero dependencies, offline.

## Features

- **3-pane terminal UI** — category sidebar · result list · markdown detail, dark theme with mono/sans mix.
- **Fuzzy search** across name (×10), description (×5), tags (×3), triggers, body text — space-separated words behave as AND.
- **Hit highlighting** with `<mark>` on every field in the result list.
- **Filters** — type toggle (skill / knowledge / all), single-click category scoping; counts update live.
- **Keyboard-first** — `/` focus search, `j`/`k` navigate, `Enter` open, `Esc` clear, `g`/`G` top/bottom, `1`/`2`/`3` type filter.
- **Chips** on the detail pane surface tags, triggers, source project, and version at a glance.
- **Self-hosted markdown renderer** — h1–3, code blocks, lists, blockquotes, links. No JS framework, no build step.
- **Zero dependencies** — two tiny Node scripts (`build-index.mjs`, `serve.mjs`) and one HTML file. No `npm install`, no bundler.

## File structure

```
skills-explorer/
├── README.md              (this file)
├── manifest.json
└── browser/
    ├── build-index.mjs    # walks .claude/skills/ + .claude/knowledge/, emits skills.json
    ├── serve.mjs          # 40-line static file server (default :4174)
    └── index.html         # single-file SPA (inline CSS + JS)
    └── skills.json        # generated (gitignored) — regenerate with the builder
```

## Usage

Run from a directory that contains `.claude/skills/` and/or `.claude/knowledge/` (e.g. any project using the skills-hub):

```bash
# 1. Build the JSON index from local skill/knowledge files
node browser/build-index.mjs
# → ✓ 217 skills + 213 knowledge → skills.json (1033.0 KB)

# 2. Serve the UI
node browser/serve.mjs
# → ▶ skills explorer → http://localhost:4174/
```

Open http://localhost:4174/ and start typing. Rebuild the index any time the underlying `.claude/` files change.

### Drop-in location

The expected layout relative to where you run the builder:

```
<cwd>/
├── .claude/
│   ├── skills/<skill-name>/SKILL.md
│   └── knowledge/<category>/<entry>.md
└── browser/               # copy this example folder here
    ├── build-index.mjs
    ├── serve.mjs
    └── index.html
```

The builder resolves `../.claude/skills` and `../.claude/knowledge` relative to itself, so placing `browser/` alongside `.claude/` is all the config it needs.

### Customization

- **Port** — `PORT=5000 node browser/serve.mjs`.
- **Theme** — tweak the CSS variables at the top of `index.html` (`--bg`, `--accent`, `--amber`, …).
- **Search weights** — edit the `matches()` function inside `index.html` to re-score fields.
- **Extra frontmatter fields** — extend the parser in `build-index.mjs` and render them as chips in `renderDetail()`.
