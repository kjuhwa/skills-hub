# Distributed Lock Visualizer

> **Why.** Animated simulation of MongoDB and Redis distributed lock patterns with 5 failure scenarios — showcases distributed locking, re-check flag, and TTL jitter skills.

## Features

- **Architecture Visualization** — Canvas-based: 2-5 service nodes, lock store (MongoDB/Redis), animated packet dots, ring pulses for acquire/fail/expire
- **MongoDB Lock** — `findAndModify`-based acquire/release with TTL expiry, real-time document state
- **Redis Lock** — `SET NX PX` + Lua release script, re-check flag verification step
- **TTL Jitter** — ±N% random jitter to prevent thundering herd, visual TTL bars with jitter range
- **5 Scenarios** — Happy Path, Deadlock (with cycle visualization), Thundering Herd (jitter on/off), Split Brain (partition zone), Retry with Backoff (exponential)
- **Event Timeline** — horizontal timeline with color-coded events per node, hover tooltips

## File structure

## Usage

Open `index.html` directly in any browser. Auto-starts Happy Path scenario.
