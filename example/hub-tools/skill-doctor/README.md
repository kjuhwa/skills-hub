# skill-doctor

> **Why.** A skills-hub working copy with 200+ entries drifts silently: a `SKILL.md` with no `description`, a knowledge entry whose folder disagrees with its `category`, two skills with the same `name`, `content.md` referenced but missing. No tool was enforcing the shape. This is a zero-dep Node CLI that lints every `SKILL.md` and `.claude/knowledge/*/*.md` and returns a non-zero exit code when errors exist — CI/pre-commit ready.

## Features

- **Frontmatter schema check** — flexible. Knowledge accepts `summary` OR `description`, `name` OR `slug` (filename fallback).
- **Folder/frontmatter agreement** — `category` must match the parent folder.
- **Duplicate name detection** — across all skills and across all knowledge separately.
- **Content completeness** — warns when `SKILL.md` is a stub with no `content.md`, or when referenced `content.md` is missing.
- **Kebab-case enforcement** — for skill names.
- **Dead cross-skill reference detection** — finds `see \`other-skill\`` prose references that don't match any real skill. Heuristic is conservative (3+ segments, >=15 chars, bounded by `see/apply/invoke/use/refer to/per`) to avoid false positives on code identifiers.
- **Two output modes** — human-readable with ANSI colors, or `--json` for CI pipelines.
- **Exit codes** — `0` clean · `1` warnings only · `2` any errors.

## File structure

```
skill-doctor/
├── README.md       (this file)
├── doctor.mjs      (single-file CLI, ~180 LOC, zero deps)
├── serve.mjs       (static + /api/run endpoint, ~40 LOC)
├── index.html      (single-file SPA UI, ~180 LOC)
└── report.json     (generated when --json is used)
```

## Usage

### CLI (primary)

```bash
# Lint the current project
node doctor.mjs

# Point at a different working copy
node doctor.mjs --root=/path/to/getskills

# CI-friendly JSON output
node doctor.mjs --json > report.json
echo "exit=$?"
```

### Web UI

```bash
DOCTOR_ROOT=/path/to/getskills PORT=4177 node serve.mjs
# → ▶ skill-doctor UI → http://localhost:4177/
```

Features:
- **status pills** — errors / warnings / info / scanned count
- **rule chips** — filter findings by severity OR by rule (click to toggle)
- **grouped by file** — all findings per file in one card
- **run button** — rescans on demand; spinner while in flight
- **clean state** — big green check when there are zero findings
- Zero frontend deps; the server is 40 LOC of Node stdlib.
