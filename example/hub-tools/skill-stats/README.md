# skill-stats

> **Why.** `skills-explorer` shows *what* is in the hub; `skill-graph` shows *how* entries relate. This shows *what the hub is made of in aggregate* — category balance, tag frequency, which source projects contributed the most patterns, how much of the corpus has `content.md` vs only `SKILL.md`. Useful for curating: are we bloated on `workflow` entries? Is `lucida-measurement` overrepresented? Which tags have become synonymous with garbage?

## Features

- **8 KPIs** — skills · knowledge · unique tags · source projects · source refs · with-content · with-triggers · avg description length.
- **Donut chart** (inline SVG) — skill category distribution with hover tooltips.
- **4 ranked bar charts** — top 20 tags, top 15 source projects, knowledge categories, top triggers. Color-coded by chart.
- **Bar rows show truncated-but-titled labels** — hover the name to see the full value.
- **Zero deps** — no Chart.js, no D3, no bundler. One HTML file with inline SVG rendering.
- **Responsive** — 2-column grid on desktop, stacks on mobile.

## File structure

```
skill-stats/
├── README.md          (this file)
├── build-stats.mjs    # frontmatter scan → stats.json (aggregates)
├── serve.mjs          # static server (default :4176)
├── index.html         # dashboard SPA (~200 LOC inline CSS+JS)
└── stats.json         # generated
```

## Usage

```bash
node build-stats.mjs
# → ✓ stats → stats.json (6.0 KB) — 217 skills, 213 knowledge, 377 unique tags, 32 projects

PORT=4176 node serve.mjs
# → ▶ http://localhost:4176/
```

### Point at a different repo

```bash
node build-stats.mjs --root=/path/to/another/skill-repo
```

### What's measured

| field | source |
|---|---|
| `categoriesSkills` / `categoriesKnowledge` | count of entries per frontmatter `category` |
| `topTags` | count across SKILL.md `tags:` arrays + knowledge `tags:` arrays |
| `topSourceProjects` | skill `source_project:` frequency |
| `topSourceRefs` | knowledge `source.ref:` prefix (split on `@`) |
| `topTriggers` | SKILL.md `triggers:` phrase frequency |
| `avgDescriptionLen` | mean character length of skill `description:` |
| `skillsWithContentMd` | how many skill folders have a `content.md` sibling |
