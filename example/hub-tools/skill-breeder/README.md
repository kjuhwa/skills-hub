# Skill Breeder

> **Why.** `skill-multiplier` runs the loop unattended and explodes the population \u2014 fun, but you can't *curate*. Breeder is the manual counterpart: you pick the parents, you inspect the draft, you decide whether it joins the pool. Accepted children become breedable with any other entry (including a prior child), so the propagation is still infinite \u2014 it's just steered by a human. Use it when you want to ask "what would X crossed with Y look like?" and iterate.

## Features

- **Two autocomplete pickers** (Parent A, Parent B) over the full 430-entry corpus, with keyboard navigation (\u2191\u2193/Enter).
- **Breed \u2192** stamps out a draft; preview shows synthesized title, gen badge, category, tag chips, and full SKILL.md body.
- **Accept / Discard / Re-roll** \u2014 accept feeds the child back into the pool as a Gen-N entry, so it becomes selectable as a future parent.
- **Random pair** shortcut for serendipity.
- **Accepted children** panel with live filter; click any row to see its full parent-chain lineage.
- **Export** dumps the accepted set (with bodies and parent chains) as JSON.
- Deterministic combiner \u2014 same `synth.mjs` engine as `skill-multiplier`, no LLM.

## Usage

```bash
# 1. Build corpus from a skills-hub checkout:
cd /path/to/skills-hub
node /path/to/skill-breeder/browser/build-index.mjs

# 2. Serve:
cd skill-breeder/browser
CORPUS=/path/to/skills.json node serve.mjs
# \u25b6 skill breeder \u2192 http://localhost:4178/
```

Corpus resolution: `$CORPUS` \u2192 `./skills.json` \u2192 `../../browser/skills.json`.
