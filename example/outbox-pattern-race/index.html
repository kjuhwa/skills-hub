Dual-Write vs Outbox Pattern

Simulate 100 transactions. Watch how many events get lost when systems fail.

Dual-Write unsafe
Sent: 0 Lost: 0 Consistency: 100%
Outbox Pattern safe
Sent: 0 Lost: 0 Consistency: 100%
How it works: Dual-Write calls DB and broker separately — a broker hiccup after DB commit causes silent data loss. Outbox pattern writes event to same DB transaction; a relay retries until broker acks. Zero loss.
