# skill-graph

> **Why.** `skills-explorer` lets you search by keyword, but the hub is also a web of *relationships* — skills that share tags, entries extracted from the same source project, patterns that cluster around specific domains. A flat list hides the clustering. This is a zero-dep browser-based force-directed graph that makes those relationships visible and navigable.

## Features

- **Force-directed SVG layout** — 430 nodes + ~860 edges, real-time Verlet simulation. No D3, no deps.
- **Two edge kinds** — shared-tag (cyan) and shared-source-project (amber). Mixed edges get both signals in their weight.
- **Smart edge pruning** — top-6 strongest connections per node so the view stays readable. Super-common tags (>40 nodes) are dropped as noise.
- **Interactive**
  - Drag any node — it sticks where you drop it until released.
  - Scroll to zoom at cursor · drag canvas to pan.
  - Click a node → side detail panel with description, chips, and clickable neighbor list.
  - `/` focuses the filter; matching nodes brighten, non-matching dim.
  - `Space` pauses the simulation; `Esc` clears selection.
- **Color = category**; **size = degree** (more connections = bigger); **outlined circles = skills**, filled = knowledge.
- **Responsive**, keyboard-friendly, works on any modern browser.

## File structure


## Usage

```bash
# 1. Build the graph from .claude/skills + .claude/knowledge
node build-graph.mjs
# → ✓ 430 nodes · 864 edges → graph.json (253.8 KB)

# 2. Serve
PORT=4175 node serve.mjs
# → ▶ http://localhost:4175/
```

### Point at a different repo

```bash
node build-graph.mjs --root=/path/to/another/getskills-clone
```

### Tuning

Edit constants at the top of `build-graph.mjs`:

- `EDGE_CAP_PER_NODE = 6` — raise for a denser graph.
- `SHARED_TAG_MIN = 2` — lower to 1 to include single-shared-tag edges (expect a lot more noise).
- The 40-node tag/source ceiling filters out super-common categories (e.g. `workflow`, `external-reference`) so the graph shows domain-specific clustering rather than everyone-connects-to-everyone.

In `index.html`, the `CATEGORY_COLOR` map controls the palette; add categories as needed.
