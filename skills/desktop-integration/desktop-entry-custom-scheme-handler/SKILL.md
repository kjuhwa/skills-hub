# Desktop Entry with Custom Scheme Handler

## When to use

Use this pattern when:
- You are packaging a Linux application that needs to handle a custom URL scheme (e.g., `myapp://open/path` or `claude://link`).
- You want other applications (browsers, terminals) to be able to open your app via `xdg-open myapp://...`.
- You are creating a `.desktop` file manually (from a build script) rather than using a package manager's automatic generation.
- You want to ensure the MIME type handler is registered immediately after package installation.

## Pattern

### Key `.desktop` file fields

```ini
[Desktop Entry]
Name=My App
Exec=/usr/bin/my-app %u
Icon=my-app
Type=Application
Terminal=false
Categories=Office;Utility;
MimeType=x-scheme-handler/myapp;
StartupWMClass=MyApp
```

### `%u` in `Exec=`

The `%u` placeholder tells the desktop environment to pass the URL as the first argument to the application when a `myapp://` link is clicked. Without `%u`, the URL is not forwarded.

### `MimeType=` format

- Format: `x-scheme-handler/<scheme>;`
- The trailing semicolon is required (the field is a semicolon-separated list).
- Multiple schemes: `MimeType=x-scheme-handler/myapp;x-scheme-handler/myapp-alt;`
- Case-sensitive: use the exact scheme string, all lowercase.

### Post-install update

```bash
update-desktop-database /usr/share/applications
```

This regenerates the `mimeinfo.cache` file that the freedesktop MIME machinery reads. Without it, the registration is not active until the database is regenerated by another mechanism (e.g., next login).

## Minimal example

```bash
#!/usr/bin/env bash
# Generate and install a .desktop file with scheme handler registration

install_desktop_entry() {
    local package_name="$1"
    local app_display_name="$2"
    local scheme="$3"             # e.g., "myapp" for "myapp://" links
    local wm_class="$4"           # e.g., "MyApp" (must match window's WM_CLASS)
    local install_dir="$5"        # e.g., "$package_root/usr/share/applications"

    mkdir -p "$install_dir"

    cat > "$install_dir/$package_name.desktop" << EOF
[Desktop Entry]
Name=$app_display_name
Exec=/usr/bin/$package_name %u
Icon=$package_name
Type=Application
Terminal=false
Categories=Office;Utility;
Comment=$app_display_name for Linux
MimeType=x-scheme-handler/$scheme;
StartupWMClass=$wm_class
EOF

    echo "Desktop entry created: $install_dir/$package_name.desktop"
}

# For AppImage: also place the .desktop inside the AppDir and
# at the top level of the AppDir (both are required)
install_appimage_desktop_entry() {
    local appdir_path="$1"
    local component_id="$2"     # reverse-DNS: io.github.org.appname
    local version="$3"
    local scheme="$4"

    local appdir_desktop="$appdir_path/${component_id}.desktop"
    cat > "$appdir_desktop" << EOF
[Desktop Entry]
Name=My App
Exec=AppRun %u
Icon=$component_id
Type=Application
Terminal=false
Categories=Network;Utility;
MimeType=x-scheme-handler/$scheme;
StartupWMClass=MyApp
X-AppImage-Version=$version
X-AppImage-Name=My App
EOF

    # Copy to usr/share/applications inside AppDir (used by appimaged)
    mkdir -p "$appdir_path/usr/share/applications"
    cp "$appdir_desktop" "$appdir_path/usr/share/applications/"
}

# Usage
install_desktop_entry \
    "my-app" \
    "My Application" \
    "myapp" \
    "MyApp" \
    "/tmp/pkg/usr/share/applications"
```

### postinst script snippet

```bash
#!/bin/sh
set -e

# Register MIME types and scheme handler
echo "Updating desktop database..."
update-desktop-database /usr/share/applications >/dev/null 2>&1 || true

# Also update icon cache if present
if command -v gtk-update-icon-cache &>/dev/null; then
    gtk-update-icon-cache -qf /usr/share/icons/hicolor 2>/dev/null || true
fi

exit 0
```

### RPM spec `%post` equivalent

```spec
%post
update-desktop-database /usr/share/applications >/dev/null 2>&1 || true

%postun
update-desktop-database /usr/share/applications >/dev/null 2>&1 || true
```

### Testing the registration

```bash
# After installation, verify the scheme handler is registered
xdg-mime query default x-scheme-handler/myapp
# Should output: my-app.desktop

# Test opening a URL
xdg-open "myapp://test/path"
```

## Why this works

### `x-scheme-handler/<scheme>` MIME type

The freedesktop.org MIME type `x-scheme-handler/myapp` is the standard way to register a URI scheme handler. When `xdg-open` encounters a `myapp://` URL, it queries `~/.config/mimeapps.list` and the system MIME database for `x-scheme-handler/myapp` and launches the associated application.

### `%u` argument substitution

The XDG desktop entry spec defines several argument substitution codes. `%u` is "A single URL" — the application will be called with the URL as a command-line argument. For scheme handlers, this passes the full `myapp://path?query=value` string to the application's process. The Electron app receives it as `process.argv[2]` (or via the `open-url` event on macOS).

### `MimeType=` trailing semicolon

The `MimeType=` field is a semicolon-separated list according to the XDG desktop entry spec. Each entry must be followed by a semicolon, including the last one. Omitting the trailing semicolon may cause some parsers to miss the last entry.

### `StartupWMClass=`

`StartupWMClass` tells the taskbar/dock which window to associate with this desktop entry. If the window's WM_CLASS property (set by Electron as the app name) does not match this field, the taskbar may show two separate icons: one for the launcher and one for the running window. The value must match what Electron sets as its application name.

### `update-desktop-database` is idempotent

Running `update-desktop-database /usr/share/applications` regenerates `mimeinfo.cache` from all `.desktop` files in the directory. It is safe to run multiple times and is fast. Always run it in `postinst` (deb) or `%post` (RPM) to ensure immediate registration.

## Pitfalls

- **`Exec=` path must be absolute** — using a relative path or a `$PATH`-based lookup (e.g., just `my-app %u`) may work in some environments but fails in others (e.g., when xdg-open invokes the handler with a clean PATH). Always use the full absolute path to the launcher.
- **AppImage `Exec=AppRun %u`** — inside an AppImage's bundled `.desktop` file, `Exec=AppRun %u` is correct (relative to AppDir). For a system-installed `.desktop` (placed by appimaged or the user), the path to the AppImage must be absolute.
- **Scheme must be lowercase** — MIME type names are case-sensitive. `x-scheme-handler/MyApp` is different from `x-scheme-handler/myapp`. Use lowercase for the scheme name. Browsers and applications typically normalize scheme names to lowercase.
- **`update-desktop-database` requires the directory path** — calling `update-desktop-database` without a path updates a different database. Always pass `/usr/share/applications` (system) or `~/.local/share/applications` (user-level).
- **Deep link forwarding in Electron** — receiving the `myapp://` URL in Electron requires listening for `app.on('open-url', ...)` on macOS or parsing `process.argv` on Linux. Electron on Linux does not fire `open-url` for scheme handlers; the URL is in `process.argv`. Plan accordingly.
