# /hub-publish-example $ARGUMENTS

Copy a local project folder (or the current working directory) into `example/<slug>/` in the `kjuhwa/skills-hub` remote, author a structured README.md, commit on a feature branch, push, and open a PR. Used after `/hub-make` (or any ad-hoc build) to make the artifact discoverable.

## Arguments

- `<slug>` — required. kebab-case. Becomes the folder name under `example/`.
- `--from=<path>` — source directory to copy (default: current cwd).
- `--title=<str>` — human title for README H1 (default: Title-cased slug).
- `--why=<str>` — one-sentence motivation; if omitted, prompt the user.
- `--stack=<csv>` — runtime/language tags (e.g. `node,html,js`) for manifest + README badges.
- `--no-pr` — push branch only, skip `gh pr create`.

## Steps

1. **Duplication check**
   - Run the logic of `/hub-list-examples` (without rendering) and stop if `example/<slug>/` already exists on `origin/main`. Suggest a new slug.
   - Also refuse if a local branch `example/<slug>` already exists in the cache.

2. **Collect metadata**
   - Resolve source directory (`--from` or cwd).
   - Determine file set: include everything under the source, but **exclude** generated/heavy artifacts by default:
     - `node_modules/`, `.git/`, `.venv/`, `dist/`, `build/`, `target/`, `*.log`, any file > 2 MB (ask before including).
     - Any path matching `.gitignore` at the source root.
   - Show the file manifest and byte totals; user confirms before copy.

3. **Prepare branch**
   - `git -C ~/.claude/skills-hub/remote fetch origin main --prune`
   - `git -C ~/.claude/skills-hub/remote checkout main`
   - `git -C ~/.claude/skills-hub/remote reset --hard origin/main`
   - `git -C ~/.claude/skills-hub/remote checkout -b example/<slug>`

4. **Copy files**
   - `mkdir -p ~/.claude/skills-hub/remote/example/<slug>/`
   - Copy approved files preserving relative paths.
   - Write `example/<slug>/manifest.json`:
     ```json
     {
       "slug": "<slug>",
       "title": "<title>",
       "stack": ["node", "html"],
       "created_at": "<ISO-8601>",
       "source_project": "<source-folder-basename>",
       "author": "<git config user.email>"
     }
     ```

5. **Generate `example/<slug>/README.md`** with sections (fill from args + inspection):
   ```
   # <title>

   > **Why.** <why — one sentence motivation>

   ## Features
   - <auto-detected or user-supplied bullets>

   ## File structure
   ```
   <tree output of the copied folder, depth 2>
   ```

   ## Usage
   <runnable commands — detect package.json / *.mjs / Dockerfile etc.>

   ## Stack
   <stack chips>

   ## Provenance
   - Generated by Claude Code via `/hub-make` + `/hub-publish-example` on <date>
   - Source working copy: `<abs-path>`
   ```

6. **Update top-level `example/README.md`** (create if missing) — append row to a catalog table so `/hub-list-examples` can read it without walking every folder.

7. **Commit + push**
   - `git add example/<slug>/ example/README.md`
   - Commit message: `example(<slug>): add <title>` (heredoc; include the `why:` line in body).
   - `git push -u origin example/<slug>`

8. **Pull request** (unless `--no-pr`)
   - `gh pr create --title "example(<slug>): add <title>" --body "$(cat <<EOF ... EOF)"`
   - Body: the same `why` + features bullets + a preview URL `https://github.com/kjuhwa/skills-hub/tree/example/<slug>/example/<slug>`.
   - Print the returned PR URL.

9. **Report**
   - Branch, commit SHA, PR URL, file count, total bytes.

## Rules

- Always confirm file manifest before copy — artifacts can silently balloon.
- Never force-push. Never write to `main` directly.
- If `gh` CLI is missing or unauthenticated, fall back to `--no-pr` and print the compare URL (`.../compare/main...example/<slug>`).
- If git `user.email` is the default `your.email@example.com`, warn before committing and offer to set a per-repo identity.
- README is the contract: without it, `/hub-list-examples` reports the entry as `[BROKEN]` and it won't be surfaced to future `/hub-make` runs.
