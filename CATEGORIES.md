# Categories

Canonical list. Keep broad (target ≤20). Use tags for fine-grained attributes.

| Category | Scope |
|---|---|
| `apm` | Application performance monitoring, observability, tracing, metrics, logging |
| `backend` | Server-side patterns, API design, middleware, request handling |
| `frontend` | UI frameworks, rendering, state management, browser APIs |
| `mobile` | iOS, Android, React Native, cross-platform |
| `devops` | CI/CD, infrastructure-as-code, deployment, containers |
| `db` | Schema design, migrations, query patterns, indexing, ORM |
| `testing` | Unit, integration, e2e, fixtures, mocking, property-based |
| `security` | Auth, crypto, OWASP, secret management, threat modeling |
| `ai` | LLM patterns, RAG, agents, embeddings, prompt engineering |
| `data` | ETL, pipelines, streaming, batch, analytics, warehousing |
| `cli` | Command-line tooling, shell scripting, argument parsing |
| `git` | Branching, hooks, rebase patterns, monorepo workflows |
| `debug` | Debugging workflows, profiling, root-cause analysis |
| `refactor` | Code transformation, migration patterns, strangler fig |
| `arch` | System design, boundaries, scaling, consistency |
| `docs` | Documentation writing, API docs, diagrams, changelog |
| `workflow` | Process/team patterns, review protocols, release flows |
| `game-dev` | Game engines, combat, RNG/gacha, progression, status effects, turn systems |
| `design` | UI component patterns, styling, theming, design tokens, design-dev handoff |
| `misc` | Doesn't fit above — revisit during `/skills_cleanup` |

## Proposing a New Category

1. Check tags first — many "new categories" are better expressed as tags.
2. If truly new, add a row to this table in the same PR as your first skill using it.
3. PR description must justify why existing categories + tags are insufficient.
